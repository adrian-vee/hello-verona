## 3. Internal Linking Plan

### Hub-and-Spoke Structure

```
Homepage (Hub)
├── → Apartments (book CTA)
├── → Location (attractions section CTA)
├── → FAQ (FAQ preview CTA)
├── → Blog (latest posts)
└── → Contact (final CTA)

Apartments
├── → Contact (booking fallback)
└── → Location (nearby attractions)

Location
├── → Guide (attractions descriptions)
├── → Apartments (book CTA)
└── → Blog (transport post)

Guide
├── → Blog (relevant posts)
├── → Location (distances)
└── → Apartments (book CTA)

Blog Posts
├── → Apartments (book CTA in every post)
├── → Guide (related content)
└── → Contact (WhatsApp CTA)
```

### Anchor Text Strategy
- Use keyword-rich anchors: "appartamento a Verona", "vicino alla Fiera"
- Vary anchors naturally: "scopri di più", "prenota ora", "vedi le distanze"
- Never repeat the exact same anchor text more than twice per page

## 7. Page Speed Optimization Notes

### Current Stack
- Next.js 14 with App Router (server components by default)
- next/image for all images (auto WebP/AVIF conversion)
- Tailwind CSS (JIT, minimal CSS)
- Google Fonts loaded via next/font (no external requests)

### Targets
- LCP < 2.5s
- FID < 100ms
- CLS < 0.1
- Lighthouse Score > 90

### Recommendations
1. Add real images via next/image with explicit width/height
2. Add priority prop to hero images
3. Consider adding `placeholder="blur"` for image loading
4. Avoid importing heavy libraries client-side
5. Test with `npm run build && npm start` then audit with Lighthouse
